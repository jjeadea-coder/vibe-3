# 공공직군 행정업무 슈퍼앱 PRD

## 1. 문서 목적

본 문서는 공공직군 행정업무 슈퍼앱의 초기 제품 요구사항을 정의한다. 구현 전 단계에서 핵심 사용자, 기능 범위, 우선순위, 데이터 요구사항, 성공 기준을 명확히 하여 프론트엔드, 백엔드, DB 설계의 기준으로 사용한다.

## 2. 제품 개요

공공직군 행정업무 슈퍼앱은 팀 단위 행정업무를 하나의 웹 애플리케이션에서 처리하기 위한 내부 업무 지원 도구다. 팀 일정 공유, 엑셀 업무 자동화, 민원 대응 챗봇, 공공 행정 뉴스 수집 기능을 제공하여 반복 업무를 줄이고 대응 품질을 표준화한다.

## 3. 목표 사용자

- 공공기관 행정 담당자
- 부서 또는 팀 단위 관리자
- 민원 대응 담당자
- 정기 보고 및 자료 취합 담당자

## 4. 핵심 목표

- 팀원의 휴가, 근무, 출장 일정을 한 화면에서 공유한다.
- 반복적인 엑셀 분리, 병합 작업을 자동화한다.
- 민원 매뉴얼 기반으로 대응 방식과 스크립트 초안을 생성한다.
- 공공 행정 관련 뉴스를 매일 아침 수집하여 업무 참고 자료로 제공한다.

## 5. 기능 요구사항

### 5.1 팀원 스케쥴 관리

#### 설명

팀원들의 휴가, 근무, 출장 등 일정을 등록하고 공유하는 캘린더 기능이다.

#### 주요 기능

- 팀원 목록 조회
- 일정 등록, 수정, 삭제
- 일정 유형 관리: 휴가, 근무, 출장, 교육, 외근, 기타
- 월간, 주간, 일간 캘린더 조회
- 팀원별 일정 필터링
- 일정 중복 또는 충돌 표시

#### 입력 데이터

- 팀원 이름
- 소속 또는 직무
- 일정 제목
- 일정 유형
- 시작일시
- 종료일시
- 장소
- 비고

#### 출력 데이터

- 캘린더 화면
- 팀원별 일정 목록
- 일정 상세 정보

#### 우선순위

MVP 필수

### 5.2 엑셀 업무 자동화

#### 설명

업로드된 엑셀 파일을 특정 컬럼 기준으로 나누거나 여러 엑셀 파일을 하나로 합치는 기능이다.

#### 주요 기능

- 엑셀 파일 업로드
- 시트 목록 조회
- 컬럼 목록 조회
- 특정 컬럼 값 기준 파일 분리
- 여러 엑셀 파일 병합
- 결과 파일 다운로드
- 처리 로그 제공

#### 입력 데이터

- `.xlsx` 또는 `.xls` 파일
- 기준 시트명
- 기준 컬럼명
- 병합 대상 파일 목록

#### 출력 데이터

- 분리된 엑셀 파일 묶음
- 병합된 단일 엑셀 파일
- 처리 결과 요약
- 오류 행 또는 누락 컬럼 정보

#### 예외 조건

- 기준 컬럼이 존재하지 않는 경우
- 파일 형식이 올바르지 않은 경우
- 빈 파일 또는 빈 시트인 경우
- 병합 대상 파일의 컬럼 구조가 서로 다른 경우

#### 우선순위

MVP 필수

### 5.3 민원 대응 챗봇

#### 설명

첨부된 민원 매뉴얼을 기반으로 민원 대응 방식, 답변 초안, 전화 응대 스크립트를 작성해주는 챗봇 기능이다.

#### 주요 기능

- 민원 매뉴얼 파일 업로드
- 매뉴얼 문서 검색
- 민원 내용 입력
- 대응 방향 추천
- 답변 문안 초안 생성
- 전화 응대 스크립트 생성
- 관련 매뉴얼 근거 표시

#### 입력 데이터

- 민원 매뉴얼 파일
- 민원 제목
- 민원 본문
- 민원 유형
- 원하는 응답 형식: 공문형, 이메일형, 전화 스크립트형

#### 출력 데이터

- 대응 요약
- 답변 초안
- 응대 스크립트
- 참고 매뉴얼 항목

#### 중요 제약

- 챗봇 답변은 최종 행정 판단이 아니라 초안으로 제공해야 한다.
- 개인정보, 민감정보 입력을 최소화하도록 안내해야 한다.
- 매뉴얼 근거가 부족한 답변은 “확인 필요”로 표시해야 한다.

#### 우선순위

MVP 이후 고도화 대상. 단, 매뉴얼 업로드와 기본 질의응답 구조는 초기 설계에 포함한다.

### 5.4 뉴스 기사 수집

#### 설명

공공 행정 관련 뉴스를 매일 아침 자동 수집하여 목록으로 제공하는 기능이다.

#### 주요 기능

- 키워드 기반 뉴스 수집
- 매일 아침 정기 실행
- 제목, 언론사, 발행일, 링크, 요약 저장
- 중복 기사 제거
- 기사 검색 및 필터링
- 중요 기사 북마크

#### 기본 키워드 예시

- 공공 행정
- 지방자치
- 민원 서비스
- 행정안전부
- 공공기관
- 디지털 정부
- 복지 행정

#### 출력 데이터

- 일자별 기사 목록
- 키워드별 기사 목록
- 기사 요약
- 원문 링크

#### 우선순위

MVP 선택. 자동 수집 스케줄러는 초기 백엔드 구조에 포함한다.

## 6. 비기능 요구사항

- 웹 기반으로 동작해야 한다.
- 프론트엔드는 TypeScript, Vite, React를 사용한다.
- 백엔드는 Python, FastAPI를 사용한다.
- Python 패키지와 가상환경 관리는 반드시 `uv`를 사용한다.
- 데이터베이스는 SQLite를 사용한다.
- 업로드 파일은 서버에서 검증 후 처리한다.
- 민원 문서와 사용자 입력에는 개인정보가 포함될 수 있으므로 저장 범위를 명확히 제한한다.
- 내부 업무망 또는 로컬 환경에서도 실행 가능해야 한다.

## 7. MVP 범위

### 포함

- 팀원 일정 CRUD
- 기본 캘린더 조회
- 엑셀 파일 업로드
- 컬럼 기준 엑셀 분리
- 엑셀 파일 병합
- SQLite 기반 데이터 저장
- FastAPI 기본 API 구조
- React 기반 화면 구조

### 제외 또는 후순위

- 조직도 연동
- 외부 그룹웨어 캘린더 연동
- 고도화된 LLM 기반 챗봇
- 기사 전문 크롤링
- 사용자 권한 체계 고도화
- 모바일 앱

## 8. 성공 기준

- 팀원이 직접 일정을 등록하고 캘린더에서 확인할 수 있다.
- 샘플 엑셀 파일을 기준 컬럼으로 분리할 수 있다.
- 여러 엑셀 파일을 하나로 병합할 수 있다.
- 민원 챗봇 기능을 위한 매뉴얼 업로드와 질의 구조가 설계되어 있다.
- 뉴스 수집 기능을 위한 스케줄러와 저장 구조가 설계되어 있다.

## 9. 주요 리스크

- 민원 챗봇 답변의 법적, 행정적 정확성 검증 필요
- 엑셀 파일 포맷 다양성으로 인한 처리 오류
- 공공기관 내부망에서 외부 뉴스 수집이 제한될 가능성
- 개인정보가 포함된 파일 업로드 시 보안 정책 필요
- SQLite 단일 파일 DB의 동시성 한계
